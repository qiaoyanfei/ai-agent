# 知库 — 微信小程序零基础指南（微信原生）

与 Flutter App **共用** 同一套 `backend` API。你不需要先学完整小程序课程，按本文 **从 0 到跑通** 即可。

---

## 零、小程序是什么（1 分钟理解）

| 概念 | 说明 |
|------|------|
| **微信小程序** | 运行在微信里的轻应用，用户 **扫码 / 搜索** 打开，不用安装 APK |
| **微信开发者工具** | 在电脑上写代码、预览、上传的 IDE（必装） |
| **AppID** | 小程序身份证；测试可用 **测试号**，上线要用注册的 AppID |
| **知库后端** | 仍是 Mac 上的 `http://IP:8000`，小程序用 `wx.request` 调接口 |

开发流程永远是：**电脑改代码 → 工具里预览 → 真机扫码试 → 满意后上传审核**。

在项目根目录（`ai-agent`）终端执行：

```bash
curl http://127.0.0.1:8000/health
```

## 三、必做配置（否则连不上后端）

### 1. 关闭域名校验（开发阶段必勾）

顶部菜单 **详情** → **本地设置** → 勾选：

- **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**

否则访问 `http://127.0.0.1:8000` 会被微信拦截。

### 2. 配置 API 地址

用编辑器打开：

`miniprogram/utils/config.js`

```javascript
const baseUrl = 'http://127.0.0.1:8000'
```

| 你怎么测 | `baseUrl` 改成 |
|----------|----------------|
| 只在电脑模拟器里点 | `http://127.0.0.1:8000`（默认即可） |
| 用自己手机微信预览 | `http://192.168.1.8:8000`（改成你 Mac 当前局域网 IP） |

查 Mac IP：**系统偏好设置 → 网络 → Wi‑Fi → IP 地址**，或终端：

```bash
ipconfig getifaddr en0
```

改完后在开发者工具点 **编译**（或保存自动编译）。

### 3. 填写 AppID（有正式号时）

打开 `miniprogram/project.config.json`，把 `"appid": "touristappid"` 改成你的 AppID。  
用测试号时也可不改。

## 四、真机调试：Network 面板怎么看（注册失败时）

真机调试连上后，开发者工具下方 **Network** 可看到手机发出的请求。

| 现象 | 含义 | 处理 |
|------|------|------|
| 请求地址是 `http://127.0.0.1:8000/...` 且失败 | 真机把 127.0.0.1 当成 **手机本机** | `utils/config.js` 设 `USE_REAL_DEVICE = true`，`DEV_LAN_HOST` 填 Mac IP |
| 红色 `register`，**Provisional headers** | 请求没真正到达 Mac 后端 | 同上 + 手机 Safari 试 `http://MacIP:8000/health` |
| Status **200**，Response 有 JSON | 接口成功 | 看业务逻辑 |
| Status **400/401** | 已到后端，业务错误 | 点该请求 → **Response** 看 `detail` 字段 |
| Console 里 `autofill` / `no permission` 红字 | 微信密码自动填充，可 **忽略** | 与注册接口无关 |

操作步骤：

1. 下方切到 **Network**，点红色 **register**（或 login）  
2. 看 **Headers** 里 **Request URL** 是否是 Mac IP，不是 `127.0.0.1`  
3. 若有状态码，点 **Response** 看返回 body  
4. 切 **Console** 看 `request.js` 或业务里打印的错误（过滤掉 autofill）

`config.js` 快速开关：

- 真机：`USE_REAL_DEVICE = true`  
- 仅模拟器：`USE_REAL_DEVICE = false`

## 五、第一次跑通（建议顺序）

1. 模拟器里应出现 **知库** 登录页  
2. 点 **注册**，输入邮箱 + 密码（至少 6 位）  
3. 成功后会跳到 **我的知识库**；可创建库、上传文档、AI 问答  
4. 若报错，看底部 **Console** 红色文字  

| 现象 | 处理 |
|------|------|
| `request:fail` / 网络错误 | 后端没开 → `docker compose up -d`；或没勾「不校验合法域名」 |
| `邮箱已注册` | 改 **登录** 用同一账号 |
| 401 / 加载失败 | 重新登录；或清空缓存：**工具 → 清缓存 → 全部清除** |
| `WXML file not found: pages/chat/chat.wxml` | ① 确认项目目录含 `miniprogram/pages/chat/chat.wxml`<br>② 应用 → 重新打开项目，选 **miniprogram** 文件夹<br>③ **清缓存 → 全部清除** 后 **编译**<br>④ 详情 → 本地设置：关闭「过滤无依赖文件」或已在本项目关闭 |

## 七、项目结构（改代码时看）

```
miniprogram/
├── app.js / app.json / app.wxss
├── project.config.json
├── utils/
│   ├── config.js       # ★ API 地址（真机 / 模拟器）
│   ├── request.js      # wx.request + Bearer
│   ├── api.js          # 业务接口封装
│   ├── chatStream.js   # 流式问答 SSE
│   └── citation.js     # 引用 [1] 解析
└── pages/
    ├── login/              # 登录 / 注册
    ├── collections/        # 知识库列表、新建
    ├── collection-detail/  # 文档上传、删除、进问答
    ├── chat/               # AI 问答、历史会话、引用跳转
    ├── preview/            # 引用来源预览（文本高亮 / PDF 打开）
    └── settings/           # 关于、退出
```

| 文件类型 | 作用 |
|----------|------|
| `.wxml` | 页面长什么样（按钮、输入框） |
| `.wxss` | 样式 |
| `.js` | 点击后做什么、调 API |
| `.json` | 页面/组件的小配置 |

每个 **页面** 四个文件同名，放在 `pages/xxx/` 下。

## 八、日常开发怎么改

1. 用 **Cursor / VS Code** 改 `miniprogram/` 里文件（和改网页类似）  
2. 保存后开发者工具会 **自动编译**（看顶部是否出现「编译成功」）  
3. 模拟器里一般会 **自动刷新**；没变就点工具栏 **编译**（或快捷键）  
4. 看 **Console** 查 `console.log` 和报错  

### 真机调试：保存代码后怎么让手机用上新版？

你截图里用的是 **真机调试**（不是「预览」二维码）。和 Flutter `hot reload` 类似，但规则略有不同：

| 你用的方式 | 保存代码后怎么做 | 说明 |
|------------|------------------|------|
| **真机调试**（工具里点「真机调试」，手机出现调试版） | 1. 保存文件 → 等 Mac 上 **编译成功**<br>2. 手机小程序通常会 **自动热重载**<br>3. 若界面没变：点工具栏 **编译**；仍不变则 **结束真机调试 → 再点一次真机调试** | 必须保持 Mac 开发者工具 **开着**、手机和 Mac **同一 WiFi** |
| **预览**（点「预览」扫二维码） | 每次改较多代码后，在工具里再点 **预览**，手机 **重新扫二维码** 打开 | 预览版不会一直连着 Mac，大改后建议重扫 |
| **体验版 / 正式版** | 工具 **上传** → 公众平台选版本 → 手机重新打开 | 上线流程，日常开发不用 |

**建议日常流程（真机调试）**：

1. Mac 打开项目 → 点 **真机调试** → 手机微信进入调试小程序（只需做一次，会话可保持）  
2. 在 Cursor 里改 `miniprogram/` → **保存**  
3. 看微信开发者工具：**编译** 是否成功；成功则看手机是否自动更新  
4. 仍显示旧界面：**编译** 按钮点一下 → 还不行就 **停止真机调试再开**  
5. 改了 `app.json`（新增页面、改首页路径）或仍异常：**工具 → 清缓存 → 全部清除**，再 **编译 + 真机调试**

**和页面里的「下拉刷新」无关**：下拉刷新是重新请求接口数据；**代码更新**靠上面的编译 / 重连真机调试。

**页面跳转**：`wx.navigateTo({ url: '/pages/chat/chat?collectionId=1' })`

## 九、和 Flutter 开发的对比（帮你迁移概念）

| Flutter | 微信小程序 |
|---------|------------|
| `Widget` | `wxml` + `wxss` |
| `dio` / `api_client` | `utils/request.js` + `wx.request` |
| `shared_preferences` | `wx.setStorageSync('token')` |
| `MaterialApp` 路由 | `app.json` 的 `pages` + `wx.navigateTo` |
| `dart-define=API_HOST` | `utils/config.js` 的 `baseUrl` |

业务逻辑仍是：**登录拿 token → 请求头带 Bearer → 调后端 REST**。

## 十、发布到微信（上线流程）

本地调试可以用 `http://192.168.x.x:8000`；**用户能搜到、能正式用的版本**必须满足微信规则，并按下面顺序做。

### 流程总览

```
准备 HTTPS 后端 → 改 config.js → 配置服务器域名 → 上传代码 → 体验版自测 → 提交审核 → 发布
```

### 在哪里配置？（入口一览）

| 要做的事 | 在哪里 | 路径 / 文件 |
|----------|--------|-------------|
| 正式 API 地址 | 本地代码 | `miniprogram/utils/config.js` → `IS_PRODUCTION`、`PROD_BASE_URL` |
| AppID | 本地 + 公众平台 | `miniprogram/project.config.json`；公众平台 **开发 → 开发管理 → 开发设置** |
| 服务器域名（HTTPS） | 微信公众平台（网页） | **开发 → 开发管理 → 开发设置 → 服务器域名** |
| 隐私指引、服务类目 | 微信公众平台 | **设置**；**设置 → 基本设置 → 服务类目** |
| 成员 / 体验者 | 微信公众平台 | **管理 → 成员管理** |
| **上传代码** | **微信开发者工具**（Mac） | 顶部工具栏 **上传** |
| 体验版 / 提审 / 发布 | 微信公众平台 | **管理 → 版本管理** |
| 开发时跳过域名校验 | 微信开发者工具 | **详情 → 本地设置** → 勾选「不校验合法域名…」（仅开发用） |

> **上传、提审、发布** 不在 Cursor 里完成：上传在 **开发者工具**，提审和发布在 **mp.weixin.qq.com** 的 **版本管理**。

### 第一步：小程序账号与 AppID

1. 登录 https://mp.weixin.qq.com/（你已注册的话跳过）
2. **开发 → 开发管理 → 开发设置** 确认 **AppID** 与 `miniprogram/project.config.json` 里 `appid` 一致（你当前为 `wx109c000e1b6615f4`）
3. **开发管理 → 成员管理**：给上传代码的微信号开通 **开发者** 权限
4. **个人主体** 小程序类目和功能会有限制；若审核要求企业资质或特定类目，需升级主体或调整介绍文案

### 第二步：后端必须公网 HTTPS（最关键）

| 要求 | 说明 |
|------|------|
| 协议 | 仅 **HTTPS**，有效证书（Let's Encrypt 等均可） |
| 域名 | 已备案域名（国内服务器常见要求） |
| 不可用 | `127.0.0.1`、`192.168.x.x`、纯 IP 端口（正式版 request 会被拦截） |

需要部署与本地相同的 **知库 backend**（Docker / 云主机均可），例如：

- `https://api.你的域名.com` → 反代到 FastAPI `:8000`
- 确认手机浏览器或 curl 能访问：`https://api.你的域名.com/health`

上传前改 `miniprogram/utils/config.js`：

```javascript
const IS_PRODUCTION = true
const PROD_BASE_URL = 'https://api.你的域名.com'  // 无末尾斜杠
```

改完后在开发者工具 **取消勾选**「不校验合法域名…」，用模拟器测一遍登录、列表、上传、问答是否正常。

### 第三步：在微信公众平台配置服务器域名

路径：**开发 → 开发管理 → 开发设置 → 服务器域名**

| 类型 | 填什么 |
|------|--------|
| **request 合法域名** | `https://api.你的域名.com`（只填域名，不带路径） |
| **uploadFile 合法域名** | 与 request **相同**（文档上传 `wx.uploadFile` 用） |
| **downloadFile 合法域名** | 同上（PDF 预览下载用） |

注意：

- 每月修改次数有限，先确认域名稳定再保存
- 域名需 **ICP 备案** 且能通过微信校验
- 开发阶段在工具里勾「不校验合法域名」仅对开发版有效，**正式用户不受影响**

### 第四步：隐私与合规（审核常卡在这里）

在公众平台补齐（按页面提示操作）：

1. **设置 → 服务内容声明 / 用户隐私保护指引**  
   说明收集邮箱、用于账号登录；若用上传文件，说明用于知识库解析与问答。
2. **开发管理 → 接口设置**  
   按实际用到的接口声明（知库以 `wx.request` / `wx.uploadFile` / `wx.downloadFile` 为主）。
3. 小程序 **服务类目** 选择与「工具 / 教育 / 办公」等相符的类目，与「AI 知识库」功能描述一致。

审核时准备：

- 1–2 个可登录的 **测试账号**（邮箱 + 密码）
- 简短说明：如何注册、建库、上传文档、提问（审核人员会点一遍）

### 第五步：上传代码（开发者工具）

1. 确认 `IS_PRODUCTION = true`，`project.config.json` 的 **appid** 正确  
2. 顶部 **上传**（不是「预览」也不是「真机调试」）  
3. 填写 **版本号**（如 `1.0.0`）和 **项目备注**  
4. 用 **管理员微信** 扫码确认  

上传成功后，代码在公众平台的 **管理 → 版本管理 → 开发版本** 里可见。

### 第六步：先体验版，再提审

1. 在 **版本管理 → 开发版本** 选刚上传的版本 → **选为体验版**  
2. **成员管理** 里体验成员扫码体验版，在真机完整走：注册 → 建库 → 上传 → 问答  
3. 无问题后：**提交审核**  
   - 上传截图（登录、知识库列表、详情、问答页）  
   - 填写功能说明与测试账号  
4. 审核通过后，在 **版本管理** 点 **发布**，全量用户才能搜到/打开正式版  

审核一般 **1–7 个工作日**；被拒会给出原因，按提示改文案或类目后重新提交。

### 第七步：上线后更新版本

以后改代码：重复 **上传 →（体验版验证）→ 提交审核 → 发布**。  
老用户会自动拉到新审核通过的版本（微信有灰度机制，一般无需用户重装）。

### 常见卡点

| 现象 | 处理 |
|------|------|
| 正式版 `request:fail url not in domain list` | 检查服务器域名是否保存成功、是否与 `PROD_BASE_URL` 完全一致 |
| 上传文件失败 | 必须单独配置 **uploadFile** 域名 |
| 审核拒：无实质服务 | 补充类目、截图、测试账号与清晰的功能说明 |
| 仍连局域网 API | 发布前务必 `IS_PRODUCTION = true` 并重新上传 |

本地开发 **不用** 先买域名；**发布前** 再完成 HTTPS 与域名配置即可。

## 十一、当前知库小程序功能（与 App 对齐）

| 功能 | 说明 |
|------|------|

**PDF**：小程序内不做页内高亮，与 App 能力差异见 `preview` 页说明。
